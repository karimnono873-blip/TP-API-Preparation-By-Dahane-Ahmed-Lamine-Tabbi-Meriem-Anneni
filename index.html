3. Analyse et Programmation : Partie 2 (Poste d'Usinage)

Le poste d'usinage est un système asynchrone nécessitant une gestion par séquence (GRAFCET). La difficulté majeure réside dans la gestion des tâches simultanées (Fraisage et Lamage) et la synchronisation avant le dégagement de la pièce.

3.1 Logique de Synchronisation en LADDER (Divergence en ET)

Pour traduire une divergence en ET (lancement simultané du fraisage et lamage), on utilise des bobines d'activation (SET) pour les étapes initiales de chaque branche :

// ÉTAPE X1 -> Divergence en ET vers X11 (Fraisage) et X21 (Lamage)
      %M1.1 (X1)    %I2.1 (s1)
-------| |-----------| |-------------------------------------( S )-- %M1.2 (X2)
                                                             ( R )-- %M1.1 (X1)
                                                             // Activation des sous-tâches
                                                             ( S )-- %M2.1 (X11)
                                                             ( S )-- %M3.1 (X21)

3.2 Gestion du Vérin Monostable (Ejection)

Contrairement aux distributeurs bistables (A, F, S) qui utilisent les instructions ( S ) et ( R ), le vérin d'éjection E nécessite un distributeur monostable. En LADDER, l'étape X4 doit maintenir la bobine standard ( ) active durant toute la phase d'éjection.

// ÉTAPE X4 : Action maintenue sur la sortie monostable E_PLUS
      %M1.4 (X4)
-------| |---------------------------------------------------( )---- %Q5.6 (E_PLUS)

4. Étapes de Configuration TIA Portal V21

Pour transférer ce code logique vers l'automate physique, respectez le protocole industriel suivant :

  1. Création du projet : Nommez le projet TP_API_AII_Dahane_Tabbi.
  2. Vue Matérielle (Hardware Configuration) :
    • Insérer un Profilé Support (Rack).
    • Emplacement 1 : Module d'alimentation (PS 307).
    • Emplacement 2 : CPU S7-300 (ex: CPU 315-2 PN/DP).
    • Emplacement 4 : Module d'entrées TOR (DI 16x24VDC).
    • Emplacement 5 : Module de sorties TOR (DO 16xRelay).
  3. Programmation Structurée : Créez les blocs fonctionnels. Utilisez l'OB1 pour appeler le FC_Cuve et FC_Usinage.
  4. Simulation S7-PLCSIM : Avant le transfert physique, cliquez sur Démarrer la simulation. TIA Portal compilera la configuration matérielle et logicielle pour vérifier l'absence d'erreurs (0 Errors, 0 Warnings).
